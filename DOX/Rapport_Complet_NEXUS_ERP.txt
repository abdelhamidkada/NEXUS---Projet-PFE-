1. PRÉSENTATION DU PROJET
================================================================================

1.1 Contexte
────────────

  NEXUS ERP est une plateforme web de gestion intégrée des ressources
  humaines (SIRH) développée dans le cadre d'un Projet de Fin d'Études
  (PFE) au sein de l'entreprise DyxIA. La plateforme vise à centraliser
  et automatiser l'ensemble des processus RH d'une organisation, depuis
  la gestion des profils employés jusqu'à l'analyse prédictive par
  intelligence artificielle.


1.2 Objectifs fonctionnels
──────────────────────────

  Le cahier des charges (CDC) définit 5 modules fonctionnels principaux :

  ┌────┬──────────────────────────────┬──────────────────────────────────────────────────────────────────┐
  │ #  │ Module                       │ Description                                                      │
  ├────┼──────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ 1  │ Core HR                      │ Gestion administrative des profils 360°, coffre-fort, compétences│
  │ 2  │ Temps & Activités (T&A)      │ Pointeuse numérique, gestion des congés, calcul des heures      │
  │ 3  │ Performance & Analytics      │ Tableau de bord KPIs, IA prédictive (attrition, formation)       │
  │ 4  │ Assistant Intelligent        │ Chatbot NLP (LangChain4j + Claude), ticketing avec SLAs          │
  │ 5  │ Authentification & RBAC      │ Sécurité JWT, 5 rôles natifs, chiffrement AES-256                │

5.2 Chiffrement AES-256-GCM
────────────────────────────

  Fichier : AttributeEncryptor.java
  Rôle    : JPA AttributeConverter transparent pour chiffrer/déchiffrer
            les données sensibles en base de données.

  - Algorithme .... : AES/GCM/NoPadding (256 bits)
  - IV ............ : Vecteur d'initialisation dynamique de 12 octets
                      (généré par SecureRandom)
  - Tag ........... : 128 bits d'authentification GCM
  - Stockage ...... : Base64(IV || CipherText) en VARCHAR(512)
  - Champs protégés : rib (Relevé d'Identité Bancaire)

  Utilisation dans l'entité EmployeeProfile :

    @Convert(converter = AttributeEncryptor.class)
    @Column(length = 512)
    private String rib;  // Chiffré de manière transparente en BDD


5.3 Hachage des mots de passe
──────────────────────────────

  - Configuration : new BCryptPasswordEncoder(12) dans SecurityConfig.java
